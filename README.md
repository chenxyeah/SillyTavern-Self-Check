# SillyTavern 写作前置自检

一个用于 SillyTavern 角色扮演的写作前置自检、预设管理、参考资料注入与临时指令插件。

> 当前版本：`0.3.0`

## v0.3.0 更新

- 参考资料库新增“批量导出”：可勾选多个资料库，一次导出为一个 `.stsc-references.json` 合集文件。
- 导入资料库时同时兼容单个资料库文件与资料库合集文件；合集只需导入一次即可全部加入。
- 新增“版本更新”页面，可直接查看当前版本、本次更新内容、远程更新状态和新版本说明。
- 插件真正升级到新版本后会弹出一次“已安装／更新”提示；检测到远程新版本时会持续提供前往扩展管理器的入口。

## v0.2.9 更新

- 修复插件可能吞掉原生 `<thinking>`、`<think>`、`<reasoning>` 或其他预设思维链内容的问题。
- 插件不再要求 `<stsc_self_check>` 位于整条回复的绝对最前面；原预设的思维链可以正常位于自检之前，自检只需保持在正文之前。
- 生成结束后只精准删除完整闭合的 `<stsc_self_check>…</stsc_self_check>`，不会删除该标签之前或之后的任何内容。
- 自检标签缺失或未闭合时不再进行破坏性裁切，避免误删思维链与正文；插件只记录格式错误并保留原消息。
- 插件不会读取、覆盖或清空 SillyTavern 消息中的原生 reasoning 数据。

## v0.2.8 更新

- 修复插件有新版本时没有及时显示更新提示的问题。旧版把检查时间保存到设置中，并设置了 6 小时检查间隔与 24 小时通知间隔，连续发布版本时可能直接跳过。
- 现在每次启动 SillyTavern 都会立即检查一次，打开插件完整管理器时也会主动复查，并在页面保持开启时每 30 分钟检查。
- 更新检测改为双重校验：同时检查本地 Git 与远程 `manifest.json` 版本；即使本地 Git 状态异常，只要远程版本号更高也能提示。
- 检测到更新后会显示不会自动消失的“插件有更新｜前往更新”提示，并在左下角魔法棒菜单的插件入口显示“更新”标记。

## v0.2.7 更新

- 内置“默认通用自检”的六道核心题现已全部改为开放问答。
- 前四道原判断题同步改写为规划式问题，要求 AI 具体说明本轮如何执行，而不是只回答“是/否”。
- 已安装旧版的用户更新后会自动迁移；仅处理内置默认预设，不影响用户自行创建的其他预设。

## v0.2.6 更新

- 上一版迁移依赖默认题全文完全一致，遇到旧版本保存产生的空格、标点或文字差异时可能跳过；本版改为按内置预设和题目标题识别。

## v0.2.5 更新

- 已安装旧版的用户更新后，会自动迁移内置“默认通用自检”中的这两道默认题；用户自行修改过的不同文本不会被强制覆盖。
- 插件会定期检查自身 Git 更新；发现新版本时显示“插件有更新”提示，点击提示可直接前往 SillyTavern 扩展管理器更新。

## v0.2.1 更新

## 核心功能

- 每轮正文生成前，强制 AI 先完成用户自定义的自检问答。
- 支持开放问答题与判断题，可设置简短、标准、详细回答。
- “提供剧情/设定依据”默认开启，可逐题关闭。
- 通用预设与角色专属预设可单独启用或叠加使用。
- 角色专属预设可绑定当前角色卡，切换角色时自动切换。
- 支持批量粘贴问题，先在临时识别页面调整，再确认导入。
- 支持自检预设和参考资料库导入、导出与分享；资料库可单独导出，也可批量打包为一个合集文件。
- 支持单次模式与双阶段严格模式。
- 自检结果只保存在插件中，下一轮 AI 读取不到上一轮自检。
- 自检格式错误时显示提示；完全跳过自检时显示红色感叹号。
- 可测试当前生效问题：AI 只回答问题并给出重复、冲突、模糊等建议，不生成正文。
- 内置参考资料库，可保存文风、强制限制和其他长期资料。
- 参考资料可自动追加成自检问题。
- 内置快捷指令库，支持临时一轮或常开，不写入聊天记录。
- 可选悬浮按钮，支持桌面和移动端拖动、查看最新自检、快速切换快捷指令，并自定义悬浮窗样式、按钮透明度与面板尺寸。
- 自动检查插件自身更新；检测到新版本后可点击提示直接前往扩展管理器，并在“版本更新”页查看版本号与更新内容。

## 安装

1. 打开 SillyTavern。
2. 进入“扩展”面板。
3. 点击“安装扩展”。
4. 粘贴仓库地址：

   `https://github.com/chenxyeah/SillyTavern-Self-Check.git`

5. 安装完成后刷新 SillyTavern。

## 两种生成模式

### 单次模式

只调用一次 API。AI 在同一轮中先输出自检，再输出正文。插件会把自检从聊天消息中剥离，只在管理器和悬浮窗中显示。

### 双阶段严格模式

调用两次 API。第一次只完成自检；第二次将本轮自检结果临时提供给 AI，再生成正文。约束更强，但会额外消耗上下文额度。

## 输出协议

若预设要求输出 thinking / reasoning，可以放在自检之前；插件只会精准剥离自检标签块：

```text
<thinking>原预设思维链（可选）</thinking>
<stsc_self_check>
<item id="问题ID">
  <answer>最终回答</answer>
  <evidence>具体剧情、角色设定或世界观依据</evidence>
</item>
</stsc_self_check>
正文、状态栏及其他正常输出格式
```

只有勾选“需要依据”的问题强制要求 `<evidence>`。缺少时插件会提示格式有误。

## 推荐隐藏正则（流式模式）

在 SillyTavern 的正则扩展中添加：

```regex
<stsc_self_check\b[^>]*>[\s\S]*?(?:<\/stsc_self_check>\s*|$)
```

替换内容留空。自检结束后，后续正文仍可继续流式显示。

## 注意事项

- 插件可以强制检查输出顺序和格式，但不能百分之百证明模型内部进行了完整推理。
- 不同模型的格式遵守能力不同。模型完全跳过自检时，插件会保留正文并显示红色警告。
- “测试预设”和“双阶段严格模式”会调用额外 API。
- 本项目仍处于测试阶段，请更新后分别测试桌面端、移动端和流式生成。
