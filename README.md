# SillyTavern 写作前置自检

一个用于 SillyTavern 角色扮演的写作前置自检、预设管理、参考资料注入与临时指令插件。

> 当前版本：`0.1.2`（界面与预设管理优化测试版）

## v0.1.2 更新

- 批量导入改为临时识别页面：先粘贴、识别、修改和删除，确认后才正式导入。
- 修复拖选或复制文字时管理器意外关闭的问题。
- 通用预设与角色预设拆分为两个独立页面。
- 新建预设改为专用弹窗，并阻止重名。
- 角色预设创建后默认不绑定，可手动绑定或换绑当前角色卡。
- 角色卡被删除后，对应预设会显示“角色卡丢失”。
- 优化平板和移动端自适应布局。

## 核心功能

- 每轮正文生成前，强制 AI 先完成用户自定义的自检问答。
- 支持开放问答题与判断题，并可设置简短、标准、详细回答。
- “提供剧情/设定依据”默认开启，可逐题关闭。
- 通用预设与角色专属预设可单独启用或叠加使用。
- 角色专属预设可直接绑定当前角色卡，切换角色时自动切换。
- 支持批量粘贴问题并自动拆分成独立问题。
- 支持单次模式与双阶段严格模式。
- 自检结果只保存在插件中，下一轮 AI 只能读取正文和状态栏，读取不到上一轮自检。
- 自检格式错误时显示提示；完全跳过自检时显示红色感叹号。
- 可测试当前生效问题：AI 只回答问题并给出重复、冲突、模糊等建议，不生成正文。
- 内置参考资料库，可保存文风、协议、限制，并设置注入位置、深度与角色。
- 参考资料可一键自动追加成自检问题。
- 内置仅下一轮生效的临时指令，不写入聊天记录。

## 安装

1. 打开 SillyTavern。
2. 进入“扩展”面板。
3. 点击“安装扩展”。
4. 粘贴本仓库地址：

   `https://github.com/chenxyeah/SillyTavern-Self-Check`

5. 安装完成后刷新 SillyTavern。

## 使用方式

安装后，在扩展设置中找到“写作前置自检”，或通过扩展菜单打开管理器。

第一次使用建议：

1. 保持“单次模式”。
2. 编辑默认通用预设中的问题。
3. 打开某张角色卡，新建一套角色专属预设并点击“绑定到当前角色”。
4. 点击“测试当前生效问题”，确认 AI 能正确理解问题。
5. 正常开始角色扮演。

## 两种生成模式

### 单次模式

只调用一次 API。AI 在同一轮中先输出自检，再输出正文。插件会把自检从聊天消息中剥离，只在管理器中显示。

### 双阶段严格模式

调用两次 API。第一次只完成自检；第二次将本轮自检结果临时提供给 AI，再生成正文。约束更强，但会额外消耗上下文额度。

## 输出协议

单次模式会要求 AI 使用以下结构：

```text
<self_check>
<item id="问题ID"><answer>最终回答</answer></item>
</self_check>
<response>
正文、状态栏及其他正常输出格式
</response>
```

插件只把 `<response>` 中的内容保留在正常聊天记录里。

## 注意事项

- 插件可以强制检查输出顺序和格式，但不能百分之百证明模型内部进行了完整推理。
- 不同模型的格式遵守能力不同。模型完全跳过自检时，插件会保留正文并显示红色警告。
- “测试预设”和“双阶段严格模式”会调用额外 API。
- 本项目仍处于首个测试版，请先在自己的 SillyTavern 中测试后再正式推广。
